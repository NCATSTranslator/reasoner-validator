check_compliance_of_trapi_response(response: Dict | None, max_kg_edges: int = 0, max_results: int = 0)ÔÉÅ

One stop validation of all components of a TRAPI-schema compliant Query.Response, including its Message against a designated Biolink Model release. The high level structure of a Query.Response is described in https://github.com/NCATSTranslator/ReasonerAPI/blob/master/docs/reference.md#response-.

The TRAPI Query.Response.Message is a Python Dictionary with three entries:

  • Query Graph (‚ÄúQGraph‚Äù): knowledge graph query input parameters

  • Knowledge Graph: output knowledge (sub-)graph containing knowledge (Biolink Model compliant nodes, edges)

    returned from the target resource (KP, ARA) for the query.

  • Results: a list of (annotated) node and edge bindings pointing into the Knowledge Graph, to represent the

    specific answers (subgraphs) satisfying the query graph constraints.

Parameters:

  • response ‚Äì Optional[Dict], Query.Response to be validated.

  • max_kg_edges ‚Äì int, maximum number of edges to be validated from the knowledge graph of the response. A value of zero triggers validation of all edges in the knowledge graph (Default: 0 - use all edges)

  • max_results ‚Äì int, target sample number of results to validate (default: 0 for ‚Äòuse all results‚Äô).

Resolve the knowledge graph node identifiers against the testcase identifier of the ‚Äòtarget‚Äô context (‚Äòsubject‚Äô or ‚Äòobject‚Äô node). If a direct match is not found for the testcase identifier, check if the nodes identifiers returned in the knowledge graph are strict ontological subclasses of the target testcase identifier (e.g. the knowledge graph may return a subclass of an instance of MONDO disease as requested by the testcase).
